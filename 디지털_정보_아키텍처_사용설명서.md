# 디지털 정보 아키텍처 사용설명서

이 문서는 `PARA + Johnny Decimal` 하이브리드 구조를 기반으로, 생성된 폴더 시스템을 바로 실무에 적용하기 위한 운영 가이드입니다.

## 1) 생성된 최상위 폴더 구조

- `000_시스템_및_수집함`
- `100_활성_프로젝트`
- `200_업무_영역`
- `300_개인_삶의_영역`
- `400_자원_및_라이브러리`
- `500_보관소`

## 2) 폴더별 운영 목적

### `000_시스템_및_수집함`
- `001_수집함_Inbox`: 임시 유입 파일 보관(주 1회 비우기)
- `002_시스템_메타데이터`: 인덱스, 라이선스, 시스템 규칙 보관
- `003_표준_작업_절차_SOP`: SOP(업무 매뉴얼) 보관

### `100_활성_프로젝트`
- 마감일/목표가 있는 진행 중 작업 보관
- 프로젝트명 규칙: `YYYY-MM-DD_프로젝트명`
- 완료 시 폴더 전체를 `500_보관소`로 이동
- 포함 템플릿: `_프로젝트_내부_표준_템플릿`
  - `_01_기획_및_행정`
  - `_02_에셋_및_소스`
  - `_03_작업_원본`
  - `_04_내보내기_최종`

### `200_업무_영역`
- 지속적으로 관리해야 하는 비즈니스 책임 영역
- 세부 카테고리:
  - `210_경영_및_기획`
  - `220_재무_및_회계`
  - `230_마케팅_및_영업`
  - `240_법무_및_행정`
  - `250_클라이언트_관리`

### `300_개인_삶의_영역`
- 개인 정보/생활 관리 영역
- 세부 카테고리:
  - `310_건강_및_의료`
  - `320_개인_재무_및_자산`
  - `330_교육_및_성장`
  - `340_가사_및_주거`

### `400_자원_및_라이브러리`
- 당장 실행하지 않지만 향후 재사용할 자료 저장
- 세부 카테고리:
  - `410_지식_창고`
  - `420_디자인_에셋`
  - `430_비디오_오디오_라이브러리`
  - `440_코드_및_IT_스니펫`

### `500_보관소`
- 완료 프로젝트 및 비활성 영역 데이터 영구 보관

## 3) 파일 명명 규칙(필수)

1. 날짜는 `YYYY-MM-DD` 형식만 사용
   - 예: `2025-04-12_계약서.pdf`
2. `최종`, `진짜최종` 같은 표현 금지
   - 대신 버전(`v1.0`, `v1.1`) 또는 날짜 사용
3. 권장 형식
   - `[날짜/버전]_[클라이언트/프로젝트명]_[에셋내용].[확장자]`
   - 예: `2025-11-22_A사디자인_로고시안_v1.0.png`

## 4) 주간 유지관리(Weekly Review, 15분)

매주 1회 아래 순서로 점검하세요.

1. `001_수집함_Inbox` 비우기
   - 필요한 파일은 적절한 상위 카테고리로 이동
   - 불필요 파일은 삭제
2. `100_활성_프로젝트` 점검
   - 완료 프로젝트를 `500_보관소`로 이동
3. 신규 프로젝트 생성
   - `YYYY-MM-DD_프로젝트명` 폴더 생성 후
   - `_프로젝트_내부_표준_템플릿`을 복사해 사용

## 5) 시작 체크리스트

- [ ] 오늘 들어온 파일은 모두 `001_수집함_Inbox`로 먼저 받기
- [ ] 새 프로젝트는 `100_활성_프로젝트`에 날짜 접두어로 생성
- [ ] 실행 중이 아닌 참고자료는 `400_자원_및_라이브러리`에 저장
- [ ] 완료된 프로젝트는 즉시 `500_보관소`로 이동
- [ ] 매주 15분, 주간 리뷰 수행
